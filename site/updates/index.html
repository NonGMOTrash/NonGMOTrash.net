i'll vent a little: in order to measure if the mix is reactable (you'll understand when u see the video) i need to have clips of both options (overhead vs low) and play them side by side so you can see when the difference appears... problem!! my recordings were always slightly desynced. i looked into my editting (wasn't the problem), the gameplay itself (was part of the problem, i was accidentally delaying some things by a few frames, throwing off the timing), but still things were desynced. grrrrr!!! as it turned out, my recording software itself (nvidia) would sometimes drop frames OR it would duplicate a frame. i tried using both davinci resolve and FFmpeg to automatically trim these duplicate frames, but both were inconsistent and i realized it wouldn't be able to detect skipped frames. so the solution... was to manually go frame by frame of every clip i use, CAREFULLY inspecting them to see if there's any errors, and manually removing duplicate frames or inserting extra frames to fill the space of where a dropped frame should have been. it's a pain, and required me referencing frame data to make sure the total durations of moves in my clip match what they should be, but i've got a good system for it now!
